# Summary: Is Best-of-N the Best of Them?

### Core Problem
Inference-time alignment aims to improve the quality of responses generated by a pre-trained base policy $\pi_{\text{ref}}$ using an imperfect reward model $\hat{r}$ as a proxy for the unknown true reward $r^*$. A common heuristic, **Best-of-N (BoN) sampling**, generates $N$ candidates and selects the one with the highest $\hat{r}$. However, BoN suffers from **reward hacking** (or overoptimization): as $N$ increases, the algorithm is more likely to select responses from the tail of the distribution where $\hat{r}$ is inaccurate, leading to high modeled rewards but poor true performance. The authors seek an algorithm that is **scaling-monotonic** (performance does not degrade as $N \to \infty$) and achieves optimal regret.

### Method: InferenceTimePessimism
The authors introduce **InferenceTimePessimism**, which implements $\chi^2$-regularization at inference time to apply the principle of "pessimism in the face of uncertainty." This separates the computational budget ($N$) from the regularization parameter ($\beta$).

**Step-by-Step Recipe:**
1. **Sampling:** Draw $N$ i.i.d. responses $\widehat{\mathcal{Y}}_{N} = (y_{1}, \ldots, y_{N}) \sim \pi_{\text{ref}}(\cdot \mid x)$.
2. **Normalization:** Compute an approximate normalization constant $\widehat{\lambda}(x)$ using a dynamic programming subroutine (`ComputeNormConstant`) such that:

$$
\frac{1}{N} \sum_{y \in \widehat{\mathcal{Y}}_{N}} \text{relu} \left(\beta^{-1} (\widehat{r}(x, y) - \widehat{\lambda}(x))\right) = 1
$$

3. **Weighting:** Define importance weights $w(y \mid x) := \text{relu} \left(\beta^{-1} (\widehat{r}(x, y) - \widehat{\lambda}(x))\right)$.
4. **Rejection Sampling:** 
   - Draw another set of $N$ responses.
   - For each response $y_i$, sample a Bernoulli variable $\xi_i$ with probability $\mathbb{P}(\xi_{i} = 1 \mid y_{i}) = \min\left\{\frac{w(y_{i}|x)}{M}, 1\right\}$, where $M := \frac{R_{\max} - \widehat{\lambda}(x)}{\beta}$.
   - Return the first $y_i$ where $\xi_i = 1$. If no response is accepted after $N$ tries, return the $(N+1)$-th sample.

### Key Formulas
*   **Coverage:** Measures the base policy's ability to generate high-quality responses:

$$
\mathcal{C}^{\pi^*}(x) := \mathbb{E}_{y \sim \pi^*(\cdot|x)} \left[ \frac{\pi^*(y \mid x)}{\pi_{\text{ref}}(y \mid x)} \right]
$$

*   **Reward Model Error:** The expected squared error under the base policy:

$$
\varepsilon_{\text{BH}}^2(x) := \mathbb{E}_{y \sim \pi_{\text{ref}}(\cdot|x)} \left[ (\hat{r}(x, y) - r^*(x, y))^2 \right]
$$

*   **Regret Bound for InferenceTimePessimism:**

$$
J(\pi^{*}; x) - J(\widehat{\pi}_{\text{Pes}}; x) \lesssim \sqrt{\mathcal{C}^{\pi^{*}}(x) \cdot \varepsilon_{\mathrm{RM}}^{2}(x)} + \beta \cdot \mathcal{C}^{\pi^{*}}(x) + \beta^{-1} \cdot \varepsilon_{\mathrm{RM}}^{2}(x)
$$

    By setting $\beta \asymp \sqrt{\varepsilon_{\mathrm{RM}}^{2}(x) / \mathcal{C}^{\pi^{*}}(x)}$, the algorithm achieves the optimal "skyline" regret of $\sqrt{\mathcal{C}^{\pi^{*}}(x) \cdot \varepsilon_{\mathrm{RM}}^{2}(x)}$.

### Key Quantitative Results
*   **Theoretical Optimality:** BoN-Alignment generally fails to reach the optimal regret skyline, scaling instead with $\varepsilon_{\text{RM}}^{2/3}(x)$. In contrast, InferenceTimePessimism is regret-optimal and scaling-monotonic.
*   **Query Complexity:** To achieve optimal regret, any sample-and-evaluate algorithm requires $N \gtrsim 1/\varepsilon_{\text{RM}}(x)$ queries; InferenceTimePessimism matches this lower bound.
*   **Empirical Performance:** Evaluated on GSM8K, MMLU, and MATH using policies like Phi-3-Mini and various reward models (OASST, GEMMA-RM, LLAMA-RM, ARMO-RM). 
    *   **Scaling:** While BoN accuracy dipped as $N$ increased (due to overoptimization), InferenceTimePessimism accuracy increased monotonically.
    *   **Accuracy Lift:** For Phi-3-Mini on MATH using LLAMA-RM, both algorithms showed significant lift ($\sim 41\%$), but InferenceTimePessimism remained robust to larger $N$.

### Stated Limitations
*   **Conservative Error Metric:** The use of $\varepsilon_{\text{BH}}^2$ (expected squared error) may be a conservative measure of reward mismatch.
*   **Binary Reward Constraints:** In tasks where $r^*$ is binary (e.g., final answer correctness), the gap between $\mathcal{C}^{\pi^*}$ and the uniform coverage $\mathcal{C}_{\infty}^{\pi^*}$ vanishes. In these specific regimes, BoN can be near-optimal, making the empirical gains of InferenceTimePessimism less statistically significant.
*   **Evaluation Scope:** The authors note that more realistic environments, such as evaluating the correctness of full proofs rather than just final answers, are required to further demonstrate the benefits of the algorithm.
