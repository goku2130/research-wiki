# Regularized Best-of-N Sampling (MBR-BoN)

### Core Problem
Best-of-N (BoN) sampling is a decoding-time alignment strategy where $N$ responses are sampled from a reference policy, and the one with the highest score from a proxy reward model is selected. However, BoN is susceptible to **reward hacking**, where the system over-optimizes for a misspecified proxy reward model that does not perfectly reflect true human preferences. This often leads to performance degradation on the true objective as $N$ increases, particularly when the reward model is trained on limited or noisy data.

### Method
The authors propose **MBR-BoN**, which mitigates reward hacking by introducing the Minimum Bayes Risk (MBR) objective as a proximity regularization term. This prevents the selected output from deviating too far from the distribution of the reference policy.

**Step-by-Step Recipe:**
1. **Sampling:** Generate a set $Y_{\text{ref}}$ of $N$ responses from the reference language model $\pi_{\text{ref}}(\cdot|x)$ for a given input $x$.
2. **Reward Scoring:** Compute the proxy reward $R(x, y)$ for each response $y \in Y_{\text{ref}}$.
3. **Utility Computation:** For every pair of responses $(y, y')$ in $Y_{\text{ref}}$, calculate a utility score $U(y, y')$. The authors use the cosine similarity of embeddings generated by `all-mpnet-base-v2`:

$$
U(y, y') = \cos(\text{emb}(y), \text{emb}(y'))
$$

4. **Regularized Selection:** Select the response that maximizes the combined score of the proxy reward and the average utility (the MBR objective), weighted by a hyperparameter $\beta$:

$$
y_{\text{MBR-BoN}}(x) = \arg\max_{y \in Y_{\text{ref}}} R(x, y) + \beta \sum_{y' \in Y_{\text{ref}}} \frac{1}{N} U(y, y')
$$

**Theoretical Justification:**
The authors analytically demonstrate that the MBR objective corresponds to minimizing the **Wasserstein Distance (WD)** between the selection strategy and the empirical distribution of the reference policy. Unlike KL-divergence, which is sensitive to minor sequence variances and requires exponential sample sizes for reliable estimation, WD utilizes semantic distance via the utility function, making it more robust for finite-sample inference.

### Key Quantitative Results
The method was evaluated using Mistral-7B-SFT-beta and Dolly-v2-3b on the AlpacaFarm and Anthropic’s hh-rlhf datasets, using Eurus as the gold reference reward model.

*   **Decoding Performance:** MBR-BoN consistently outperformed vanilla BoN and pure MBR decoding across various proxy reward models (SHP-Large, SHP-XL, and OASST).
*   **Hyperparameter $\beta$:** The optimal $\beta$ varies by dataset and reward model. For AlpacaFarm, $\beta=0.5$ was optimal for SHP-Large/XL, while $\beta=20.0$ was optimal for OASST. The authors found that as few as 10 instances in a development set were sufficient to tune $\beta$ to a near-optimal value.
*   **Preference Dataset Generation:** MBR-BoN was used to generate pairwise preference datasets (chosen = MBR-BoN selection; rejected = lowest reward response). Models trained via Direct Preference Optimization (DPO) on MBR-BoN datasets outperformed those trained on vanilla BoN datasets.
*   **Correlation:** Empirical tests showed a strong negative correlation between the distance from the center of the sample distribution (measured via PCA/ICA) and the MBR objective, confirming its role as a proximity regularizer.

### Limitations
*   **Computational Complexity:** MBR-BoN inherits the quadratic computational complexity of MBR decoding relative to the number of samples $N$ due to the pairwise utility calculations.
*   **Evaluation Metrics:** The study relied on automated evaluators (Eurus and GPT-4o) rather than direct human evaluation.
*   **Algorithm Scope:** The application to preference learning was limited specifically to the DPO algorithm; other preference optimization algorithms were not evaluated.
