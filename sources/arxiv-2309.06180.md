High-throughput serving of autoregressive large language models (LLMs) is fundamentally memory-bound due to the dynamic growth of the key-value (KV) cache. Existing systems pre-allocate contiguous GPU memory per request based on maximum sequence length, causing severe internal and external fragmentation that limits batch size. Furthermore, contiguous allocation prevents memory sharing across sequences generated by advanced decoding algorithms. To resolve this, the authors introduce *PagedAttention*, an algorithm inspired by operating system virtual memory. The method partitions the KV cache into fixed-size logical blocks mapped to non-contiguous physical GPU pages via a centralized block table. Physical blocks are allocated dynamically as tokens are generated. Memory sharing is enabled through reference counts and a block-level copy-on-write mechanism. Under memory pressure, vLLM employs an all-or-nothing eviction policy, supporting either swapping blocks to CPU RAM or recomputing them, while a centralized scheduler broadcasts block tables to distributed GPU workers to ensure consistent mapping without inter-worker synchronization.

The autoregressive generation process factorizes sequence probability as $P(x) = P(x_1) \cdot P(x_2 \mid x_1) \cdots P(x_n \mid x_1, \dots, x_{n-1})$. Standard self-attention computes query, key, and value vectors via $q_i = W_q x_i, \ k_i = W_k x_i, \ v_i = W_v x_i$, followed by attention scores and outputs: $a_{ij} = \frac{\exp(q_i^T k_j / \sqrt{d})}{\sum_{t=1}^i \exp(q_i^T k_t / \sqrt{d})}, \ o_i = \sum_{j=1}^i a_{ij} v_j$. PagedAttention reformulates this into block-wise operations: $A_{ij} = \frac{\exp(q_i^T K_j / \sqrt{d})}{\sum_{t=1}^{\lceil i/B \rceil} \exp(q_i^T K_t 1 / \sqrt{d})}, \quad o_i = \sum_{j=1}^{\lceil i/B \rceil} V_j A_{ij}^T$, where $B$ is the KV block size and $K_j, V_j$ denote the key and value vectors within block $j$.

Evaluations on OPT and LLaMA models demonstrate that vLLM achieves 2–4× higher throughput than state-of-the-art systems like FasterTransformer and Orca at equivalent latency levels. Profiling reveals that existing systems waste 61.8%–79.6% of KV cache memory, utilizing only 20.4%–38.2% for actual token states. On the ShareGPT dataset, vLLM sustains 1.7×–2.7× higher request rates than Orca (Oracle) and 2.7×–8× than Orca (Max). Memory sharing yields 6.1%–9.8% savings for parallel sampling and 37.6%–55.2% for beam search on Alpaca, with even higher gains (up to 66.3%) on ShareGPT. Shared prefix scenarios improve throughput by 1.67× to 3.58×. The system defaults to a block size of 16 tokens to balance GPU utilization and fragmentation.

The PagedAttention kernel incurs 20%–26% higher attention latency than highly optimized contiguous implementations due to block table lookups and branching overhead. Swapping evicted blocks to CPU RAM becomes inefficient with small block sizes due to PCIe bandwidth saturation, though recomputation remains viable and never exceeds 20% of swapping's latency. The authors note that this paging approach is specifically optimized for LLM serving's dynamic memory requirements; applying it to compute-bound or static-shape workloads may degrade performance due to memory indirection overhead. Additionally, swapping preemptions temporarily halt the acceptance of new requests until preempted sequences complete.
