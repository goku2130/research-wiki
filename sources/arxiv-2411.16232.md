**Core Problem**
Modern wireless communication infrastructures face escalating complexity driven by 5G, IoT, edge computing, and SDN/NFV integration, rendering traditional reactive management approaches insufficient. The authors address the critical need for proactive, real-time decision-making in mission-critical infrastructureless Wi-Fi mesh networks, where continuous, secure, and reliable communication supports essential services like smart healthcare and first responders. While Large Language Models (LLMs) exhibit strong zero-shot reasoning and generalization, their capacity to function as autonomous planners for network configuration remains underexplored. This work benchmarks pre-trained LLMs as zero-shot learners to evaluate their viability for generating real-time network management recommendations.

**Methodology**
The experimental protocol follows a structured zero-shot prompting pipeline. First, a three-node mesh network scenario is established, where nodes periodically transmit status reports containing TX/RX throughput, latency, packet loss, neighbor lists, and security events such as channel interference or jamming. Second, a system prompt initializes the LLM as a network monitoring expert, defining the operational context, network parameters (e.g., starting on channel 36), and a comprehensive set of valid actions. These actions include disconnecting nodes, switching channels (36–46), updating neighbor lists, adjusting network throughput (0.1, 2, or 10 Mb/s), updating local node coordinates, or executing no action. Third, a user prompt delivers specific network observations alongside strict formatting constraints: the model must select exactly one action, wrap the selection in `<ACTION>chosen action</ACTION>` tags, and refrain from generating any additional text. Fourth, the LLM processes the observations via zero-shot reasoning to output a configuration recommendation. Finally, generated responses are evaluated against ground-truth labeled responses using ROUGE-1, METEOR, and BLEU metrics. The study compares GPT-3.5, GPT-4 Turbo, Llama series, Falcon series, Phi3-3.8B Mini-4K, and Falcon-Mamba-7B. The source does not present mathematical formulations; evaluation relies exclusively on prompt engineering and standard NLP similarity metrics. Prompt sensitivity is also tested by varying trailing newline characters in the user prompt.

**Quantitative Results**
GPT-3.5 and GPT-4 Turbo consistently produce high-quality zero-shot responses, attributed to extensive pre-training and large-scale knowledge bases. Most open-source models underperform due to smaller pre-training scales, though Phi3-3.8B Mini-4K and Falcon-Mamba-7B demonstrate notably competitive performance. For Falcon-Mamba-7B, metric scores exhibit pronounced sensitivity to prompt formatting. When the prompt ends with a single newline (`\n`), the model achieves ROUGE-1 = 0.82, METEOR = 0.78, and BLEU = 0.58. Removing the newline drops performance to ROUGE-1 = 0.38, METEOR = 0.38, and BLEU = 0.54. Using double newlines (`\n\n`) yields ROUGE-1 = 0.67, METEOR = 0.66, and BLEU = 0.51. The consistent pattern of high ROUGE-1 and METEOR scores alongside moderate BLEU scores across several open-source models indicates that these architectures grasp the core task and contextual constraints but struggle with exact-match precision relative to the labeled responses.

**Limitations**
The study identifies several operational and technical constraints. Commercial GPT models, despite superior zero-shot performance, are hindered by restricted API access, high usage costs, and data privacy concerns. Open-source alternatives exhibit pronounced sensitivity to minor prompt structural variations, such as trailing whitespace, which drastically alters evaluation metrics. Furthermore, the reliance on ROUGE/METEOR/BLEU metrics against static labeled responses may not fully capture functional correctness, real-time operational safety, or dynamic network state transitions. The authors conclude that while open-source LLMs show promising zero-shot capabilities, they currently lack the reliability and resiliency required for mission-critical wireless management. Consequently, targeted fine-tuning, alignment, and robust prompt engineering are necessary before these models can be safely deployed in autonomous network management systems.
