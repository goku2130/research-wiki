# Rubric-Based Rewards for Reinforcement Learning

## Core Problem
Recent advancements in Large Language Model (LLM) reasoning are largely attributed to Reinforcement Learning with Verifiable Rewards (RLVR). RLVR utilizes deterministic, rule-based verifiers (e.g., unit tests for code or exact string matching for math) to provide reliable reward signals that are resistant to reward hacking, enabling stable training at scale. However, RLVR is confined to domains with automatically checkable outcomes. 

Many high-value applications—such as creative writing, long-form question answering, and scientific reasoning—operate in subjective, non-verifiable domains where correctness cannot be captured by binary signals. While Reinforcement Learning from Human Feedback (RLHF) can address these domains using learned reward models, it suffers from several drawbacks: the requirement for massive volumes of preference data, a lack of granular control over alignment criteria, and a high susceptibility to reward hacking (e.g., the model exploiting verbosity bias).

## Method: Rubric-Based RL
Rubric-based rewards aim to extend the scalability and reliability of RLVR to non-verifiable domains by decomposing desired model behavior into structured, interpretable criteria.

### Step-by-Step Recipe
1.  **Rubric Creation**: Develop a detailed scoring prompt (a rubric) that specifies the evaluation process. These rubrics can be:
    *   **General**: Applied across a domain.
    *   **Prompt/Instance-Specific**: Tailored to a specific prompt-response pair, often synthetically generated by an LLM or written by domain experts (e.g., physician-written rubrics in HealthBench).
2.  **Criterion Decomposition**: Structure the rubric as a checklist of distinct quality dimensions. Each criterion captures a single attribute the response should be rewarded or penalized for.
3.  **Weight Assignment**: Assign weights to each criterion to define its relative importance in the final reward calculation.
4.  **LLM-as-a-Judge Evaluation**: Use a foundation model as a judge to evaluate the candidate response against the rubric. To improve accuracy and interpretability, the judge is typically prompted to use **Chain-of-Thought (CoT)**, providing a step-by-step explanation before the final score.
5.  **Reward Derivation**: The judge evaluates each criterion (often yielding a binary pass/fail score). These individual scores are then aggregated based on their assigned weights to produce a multi-dimensional reward signal.
6.  **RL Optimization**: This aggregated reward is used as the signal for online RL algorithms (such as PPO or GRPO) to update the model's policy.

## Key Quantitative Results
The provided text does not list specific numerical performance gains; however, it notes the following qualitative findings:
*   **Reliability**: Rubrics significantly improve the reliability of LLM evaluation compared to single, general prompts.
*   **Human Agreement**: In the MultiChallenge benchmark, prompt-specific rubrics were found to improve the agreement between expert humans and LLM judges.
*   **Scalability**: LLM-as-a-Judge is described as a scalable way to approximate human preferences, which are otherwise expensive to obtain.

## Limitations
The primary limitation of this approach is that the reward signal depends on an LLM judge, which is an imperfect proxy for human judgment and introduces several biases:
*   **Position Bias**: Favoring outputs based on their order in the prompt.
*   **Verbosity Bias**: Assigning higher scores to longer responses.
*   **Self-enhancement Bias**: Favoring responses generated by the judge itself.
*   **Capability Bias**: Difficulty evaluating responses to prompts the judge cannot solve.
*   **Distribution Bias**: A tendency to cluster scores around specific values (e.g., mostly 3s on a 1-5 scale).
*   **Prompt Sensitivity**: LLM judges are highly sensitive to prompt details, requiring careful calibration, tuning, and the collection of high-quality human labels to align with human judgment.
