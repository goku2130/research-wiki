The ReAct paradigm addresses the fundamental disconnect between reasoning and acting in large language models (LLMs). While chain-of-thought (CoT) prompting enables multi-step reasoning, it operates as a static black box prone to hallucination and error propagation. Conversely, action-only approaches lack the abstract reasoning and working memory required to track goals, decompose tasks, or handle environmental exceptions. ReAct resolves this by synergizing verbal reasoning traces with task-specific actions, allowing the model to dynamically induce, track, and adjust action plans while interfacing with external environments.

The ReAct protocol follows a structured recipe. First, the agent’s action space is augmented to $\hat{\mathcal{A}} = \mathcal{A} \cup \mathcal{L}$, integrating domain actions $\mathcal{A}$ with language thoughts $\mathcal{L}$. The model operates under a policy $\pi(a_t | c_t)$, where the context $c_t = (o_1, a_1, \dots, o_{t-1}, a_{t-1}, o_t)$ accumulates the interaction history. At each step, the LLM generates a thought $\hat{a}_t \in \mathcal{L}$ to update the context via $c_{t+1} = (c_t, \hat{a}_t)$ without triggering environmental feedback. Thoughts serve to decompose goals, extract relevant information, apply commonsense, and track progress. Subsequently, a concrete action $a_t \in \mathcal{A}$ is executed to query an environment (e.g., a Wikipedia API with `search[entity]`, `lookup[string]`, and `finish[answer]` commands). New observations $o_{t+1}$ are returned and appended to the context, closing the loop. For implementation, the method uses few-shot in-context prompting with frozen LLMs, where human-annotated trajectories demonstrate the interleaved Thought-Action-Observation format. To balance internal knowledge and external grounding, the authors implement heuristic switching: ReAct reverts to CoT with self-consistency (CoT-SC) when step limits are exhausted, while CoT-SC delegates to ReAct when internal confidence drops below a threshold. Finally, for scaling, correct ReAct trajectories can bootstrap supervised finetuning on smaller models.

Quantitative evaluations demonstrate ReAct’s efficacy across four benchmarks. On HotpotQA, ReAct achieves an exact match (EM) score of 27.4, while the combined CoT-SC → ReAct method reaches 35.1, compared to 67.5 for supervised state-of-the-art. On FEVER, ReAct attains 60.9% accuracy, with CoT-SC → ReAct reaching 62.0% against an 89.5% supervised baseline. In interactive decision-making, ReAct outperforms imitation and reinforcement learning baselines trained on $10^3 \sim 10^5$ instances by an absolute 34% success rate on ALFWorld (71% vs. 45% for Act-only) and a 10% absolute gain on WebShop (40.0% vs. 29.1% for IL). Human analysis of HotpotQA failures reveals that CoT suffers from hallucination in 56% of failure cases, whereas ReAct’s failures stem primarily from reasoning errors (47%) and non-informative search results (23%). Finetuning experiments further show that PaLM-62B models trained on ReAct trajectories surpass all PaLM-540B prompting methods.

Despite its advantages, ReAct exhibits several limitations. The prompting paradigm is constrained by input length, requiring extensive demonstrations for complex tasks with large action spaces. Greedy decoding can cause the model to enter repetitive reasoning loops, failing to recover from uninformative search results. The method’s reliance on external APIs means that poor retrieval directly derails the reasoning trajectory. Furthermore, scaling ReAct via finetuning demands high-quality human annotations, and hooking LLMs to unrestricted external environments introduces safety risks that require careful action-space design.
