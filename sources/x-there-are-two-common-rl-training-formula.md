The author, Cameron R. Wolfe, Ph.D., discusses two common Reinforcement Learning (RL) training formulations for Large Language Models (LLMs): the Markov Decision Process (MDP) formulation and the bandit formulation.

**Core Problem:**
LLMs generate output token by token, which can be framed as a sequential decision-making process. The core problem is how to effectively apply RL to train LLMs, considering their autoregressive nature and the typical reward mechanisms. Specifically, how to assign credit for rewards, especially when rewards are only provided at the end of a generated sequence (outcome supervision), rather than at each token generation step.

**Method/Recipe Step by Step:**

1.  **Recall LLM Generation:** An LLM generates output by predicting tokens sequentially. Each token prediction can be viewed as an individual action in an RL setup.

2.  **MDP Formulation:**
    *   **Modeling:** Each token in the LLM's output is modeled as an individual action within a Markov Decision Process (MDP). An MDP is a probabilistic framework comprising states, actions, transition probabilities, and rewards.
    *   **Initial State:** The prompt given to the LLM.
    *   **Policy:** The LLM's mechanism for predicting tokens.
    *   **Actions:** Selecting a token from the LLM's predicted token distribution. Each token is an action.
    *   **State Transition:** After a token is predicted, it is added to the current state, and this updated state is used by the LLM to predict the next token (autoregressive prediction).
    *   **Episode End:** The LLM predicts a stop token (e.g., `<|end_of_text|>` or `<eos>`), completing the generation and yielding a full trajectory.
    *   **Reward Mechanism (Conceptual vs. Actual):** Conceptually, in an MDP, a reward is provided at every step. However, for most LLMs, training uses "outcome supervision," meaning a reward is assigned only after the model generates a complete response (i.e., after the `<eos>` token). This raises a credit assignment problem for individual token actions.

3.  **Bandit Formulation:**
    *   **Motivation:** Addresses the credit assignment problem in outcome supervision settings where rewards are sparse (only at the end). Instead of modeling each token as an action, the entire response is considered a single action.
    *   **Modeling:** Inspired by the contextual bandit problem. The agent (LLM) chooses one action, receives a reward, and the episode ends.
    *   **Action:** The full completion generated by the LLM for a given prompt.
    *   **Reward:** An outcome reward received for the entire completion.
    *   **Trajectory:** The complete trajectory consists of a single action and its corresponding reward.

The source does not provide specific quantitative results or numbers. It states that both formulations are viable and used, and that the bandit formulation "tends to be more simple (and has been argued to perform similarly despite this simplicity)."

**Stated Limitations:**
*   **MDP Formulation with Outcome Supervision:** The primary limitation is the "credit assignment" problem when using outcome supervision (rewards only at the end of a sequence) with an MDP formulation that conceptually expects rewards at every step.
*   **General Applicability:** While both formulations are viable, the bandit formulation is noted as "quite fitting for LLMs" given their usual outcome reward structure, and it is simpler. However, the MDP formulation is also "commonly used." The source does not state that one is universally superior, but rather that their suitability depends on the reward structure and specific RL algorithm (e.g., PPO uses MDP, REINFORCE and RLOO adopt bandit).
